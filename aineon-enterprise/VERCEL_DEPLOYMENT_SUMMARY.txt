================================================================================
                         KEY DECISIONS MADE
================================================================================

ARCHITECTURE CHANGE:
  FROM: Container-based (Render) + WebSocket
  TO:   Serverless functions (Vercel) + HTTP polling

REAL-TIME UPDATES:
  FROM: WebSocket (port 8765, <10ms latency)
  TO:   HTTP polling (3-5 second intervals)
  WHY:  Vercel doesn't support persistent WebSocket connections

STATIC ASSETS:
  FROM: Bundled in main FastAPI application
  TO:   Served from /public/ directory by Vercel CDN
  WHY:  Vercel automatically serves /public/ as static content

DEPLOYMENT:
  FROM: 2-5 minutes build + start
  TO:   30-60 seconds build + deploy
  WHY:  Serverless = faster deployment & no container overhead

COST:
  FROM: $7/month minimum
  TO:   Free tier + pay-per-use
  WHY:  Serverless pricing matches actual usage

================================================================================
                       POLLING IMPLEMENTATION
================================================================================

Frontend (Dashboard):
  Endpoint: /api/profit/live
  Interval: 3 seconds (POLL_INTERVAL = 3000ms)
  Method: GET (stateless)
  Response: JSON with latest metrics

Backend (Serverless):
  Function: get_live_profit()
  Location: api/index.py:154-162
  Returns: Profit metrics with timestamp
  No state maintained

Data Flow:
  1. Dashboard polls every 3 seconds
  2. Sends GET request to /api/profit/live
  3. Function returns latest metrics
  4. Dashboard updates UI
  5. Repeat cycle

Advantages:
  ✅ Stateless (fits serverless model)
  ✅ Simple HTTP (no special protocol)
  ✅ Scales automatically
  ✅ Works in any browser
  ✅ No connection management

Trade-off:
  ⚠️ 3-5 second latency (vs <10ms WebSocket)
  - Acceptable for dashboard use case
  - Real-time enough for monitoring
  - No perceptible difference for users
